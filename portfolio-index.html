Approach

How I work

Every engagement starts with a performance problem. The design follows the gap — not the content list, not the stakeholder's first suggestion, and not what was done last time.
Performance consulting
Gap analysis before design. Environment, motivation, and skill gaps treated separately. Training is rarely the first answer.
Interactive prototyping
HTML, CSS, and JS used to build working artefacts — not mockups. Stakeholders see and feel the experience, not a slide deck.
Simulation design
Decision-based scenarios with designed failure states. Learning happens when people make real choices with real consequences in a safe environment.
Measurement
Behaviour change metrics, not completion rates. Baseline before day one. Delta at day 30. The design is accountable for the outcome.
Scalability
Every programme is designed to be handed off. Facilitator guides, async modules, train-the-trainer — built in from the start, not added later.
AI-assisted design
Claude and other AI tools used throughout the design process — for rapid prototyping, scenario generation, and interactive artefact development.
